# 商业竞品共识稿

## 结论先行

1. 真正卖得好的，不是“会写代码”的能力本身，而是围绕它形成的完整购买包：
   `定价清晰 + 团队版 + 企业版 + 安全合规 + 部署路径 + 可展示的 ROI`。
2. OpenHands 最应该研究的不是单个模型效果，而是：
   `从自治 agent 能力，如何转成企业可采购、可治理、可续费的产品形态`。
3. 直接同层最强对照是 `Devin / Cursor / Windsurf`；
   最值得模仿的企业包装样本是 `Augment / Cline / Dify`；
   最值得警惕的相邻层爆款是 `Replit Agent / Lovable`；
   中国市场最值得拆解的是 `Trae / 通义灵码 / 文心快码`。

## 共识判断

### 已证明能卖

- `Cursor`：个人订阅、团队协作、企业控制台三段式已经非常成熟。
- `Devin`：高价 seat 包装已经成立，且融资和市场心智领先。
- `Windsurf`：本地 IDE + 云 agent 的组合形态已经成型。
- `Replit Agent`：从“AI 开发工具”扩成“AI 构建与部署平台”，非工程预算也能买。
- `Dify`：开源分发 + 企业回收 + Azure Marketplace 是非常清晰的 B2B 路线。
- `Lovable`：把“全民做软件”卖成了高速增长的消费级/团队级产品。
- `Trae / 通义灵码 / 文心快码`：中国市场已经证明“免费切入 + 企业收费 + 私有化与合规”是有效打法。

### 值得 OpenHands 立即模仿

- 企业版包装写清楚：`SSO / SCIM / 审计日志 / VPC / 私网 / 白名单 / 专有部署`
- 把“agent 能力”拆成可采购模块：`代码生成 / 代码审查 / 终端代理 / 团队知识 / 企业治理`
- 建立公开客户与 ROI 证据库：不要只讲 benchmark，要讲 `席位数 / 使用率 / 时间缩短 / 成本下降`
- 多入口产品形态：`Web GUI + IDE/CLI + API + 集成入口`

### 不值得盲抄

- 只靠“最强 agent”“最强 benchmark”的宣传
- 只强调 vibe coding，不补部署、安全、采购与治理
- 大量依赖品牌与资本热度，但公开客户证据薄弱的叙事
