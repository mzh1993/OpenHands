OpenHands Strategy Lens

不是谁最会写代码，而是谁最会把 AI 开发能力卖成产品

这轮调研的结论很明确：市场已经不再为“一个会写代码的模型”单独付钱， 真正能卖出去的是一整套可采购、可治理、可部署、可证明 ROI 的产品包装。 OpenHands 现在最需要补的，不是概念，而是把自治软件工程能力翻译成企业能买、能试点、能续费的商品。

市场已经验证了什么

成功样本跨国家、跨形态，但商业规律高度一致。它们卖出去，不是因为“更像科幻”，而是因为更像真正能进采购流程的生产工具。

1. Seat 生意已经成立

Devin、Cursor、Windsurf、Replit、Trae、通义灵码都已经用清晰的个人版、团队版、企业版定价证明：AI 开发产品可以像传统开发工具一样按席位售卖。

2. 企业买的是可控性

SSO、SCIM、Audit Logs、VPC、IP 白名单、专属部署、Marketplace、Invoice/PO，这些不再是锦上添花，而是成单条件。

3. 买单人群在变宽

Replit Agent 和 Lovable 说明，预算不只来自研发部门。产品、运营、支持、创新团队也在为 AI 开发能力买单。

10 个正式样本

这里不是按技术架构分类，而是按“OpenHands 需要从它身上学什么”来展示。每张卡都只保留 4 类信息：卖什么、卖给谁、卖得怎么样、该学什么。

P0 直接竞争 Devin

Devin

卖的是“云端自治软件工程师”。重点不是 IDE，而是把规划、执行、审查、知识和团队协作打成一个高单价 seat 产品。

  • 公开定价：Free / Pro / Max / Teams
  • 媒体报道估值高，市场心智已成型
  • 最值得学的是如何把“agent 像员工”翻译成可采购包装
P0 直接竞争 Cursor

Cursor

卖的是 AI 原生 IDE 主工作台。真正强的不是补全，而是从 Pro 到 Teams 到 Enterprise 的完整升级链路。

  • Pro $20/月，Teams $40/用户/月
  • 企业页直接写 SSO、SCIM、Audit、Admin
  • 说明 AI IDE 已经不是工具实验，而是组织级软件采购品
P0 直接竞争 Windsurf

Windsurf

卖的是本地 IDE 与云端 agent 的双栈体验。它的价值在于：告诉市场“前台编辑器 + 后台 agent”是一条自然路径。

  • Pro $20/月，Teams $40/用户/月
  • 官方公开 1M+ 用户与 4,000+ 企业客户
  • 值得 OpenHands 研究“前台入口”如何补强
P1 企业包装参考 Augment

Augment

卖点不是“更聪明”，而是“更懂大代码库”。这是很典型的企业叙事：上下文引擎、审查、策略、长仓库理解。

  • Standard $60/月/开发者
  • 强调大型代码库、长期维护、复杂工程
  • 最值得学的是如何把“代码库理解”卖成主价值
P1 OSS 路线参考 Cline

Cline

典型的开源分发 + 企业回收路线。它提醒 OpenHands：BYOK、可控推理、代码不出环境，本身就是核心卖点。

  • 个人免费，企业版走销售
  • SSO、RBAC、遥测与配置治理明确
  • 对 OpenHands 的启发非常直接
相邻层爆款 Replit Agent

Replit Agent

它卖的已经不是 coding agent，而是“从描述到 app 到部署”的完整平台。最强的启发是：预算不只在工程部门。

  • Core $20/月，Teams $35/用户/月
  • 有云市场、有客户案例、有跨职能使用故事
  • 证明 agent 可以卖给更多非工程买家
企业中台型 Dify

Dify

这是最清楚的“开源分发 + 企业回收 + 合规 + Marketplace”样本。虽然不专注 coding，但产品化路径很适合拆给 OpenHands 看。

  • Professional $59/workspace/月
  • Self-host、LTS、SOC 2、ISO 27001、Azure Marketplace
  • 告诉 OpenHands 企业版该怎么写、怎么卖
高增长相邻层 Lovable

Lovable

最值得研究的不是它有多会做 vibe coding，而是它怎么把增长、项目量、估值和企业能力一起讲成一个商业故事。

  • Pro $25/月，Business $50/月
  • 公开 ARR、项目量、融资估值与企业背书
  • 最值得学的是“商业证明可视化”
中国市场强样本 Trae

Trae

Trae 已经把中国市场最有效的打法写得很明白：免费切入、价格激进、企业治理能力显性化、形态统一。

  • 个人免费，企业从 ¥49/人/月起
  • 公开 600 万+ 注册与 100 万+ MAU 信号
  • 最值得学的是价格与企业页面语言
中国市场强样本 通义灵码

通义灵码

它的价值不在“最酷”，而在“阿里云 ToB 渠道 + 专属版 + 私有化 + ROI 案例库”已经很成熟。

  • 企业标准版 ¥79/人/月，专属版 ¥159/人/月
  • 官方称服务超万家企业、百万开发者
  • 最值得学的是企业案例库与三段式包装

第二版矩阵：观察层样本

第二版矩阵把候补样本拉进来，但用途不是重新争主样本席位，而是回答“哪些产品现在还不该升格、但必须继续盯着”。这里看的不是综合强弱，而是它们最值得 OpenHands 观察的商业信号。

强证据但被主样本覆盖

文心快码最典型。它不是弱，而是更偏中国政企深耕，和 Trae / 通义灵码放在一起时，国际可比性没那么高。

产品成熟，但不在主战场

Continue 很值得拆，但它更像 AI quality layer。它提醒 OpenHands：质量门禁可以是独立产品层，不一定都要做成自治工程师。

热度或包装强，证据还不够厚

Manus 和 Bolt 都属于这一类。一个强增长叙事，一个强 enterprise packaging，但都还缺更密集、可持续的公开客户和经营证据。

OpenHands 现在最该做什么

下面 6 件事，不是“可以考虑”，而是这轮调研后最值得优先推进的动作。重点不是再发一篇愿景文章，而是把产品和销售语言补齐。

P0

把企业版买点写清楚

  • SSO / SCIM / RBAC / Audit logs
  • Self-host / VPC / Private deployment
  • Invoice / PO / Marketplace
P0

建立公开 ROI 证据库

  • 客户案例
  • 使用率
  • 时间节省
  • 可替代的任务类型
P0

明确产品梯度

  • Community / Pro / Team / Enterprise
  • 哪些功能属于团队版
  • 哪些功能属于企业版
P1

补多入口一致性

  • Web GUI
  • CLI
  • IDE/Editor integration
  • API / SDK
P1

强化代码库理解叙事

  • 长任务
  • 跨文件修改
  • 真实执行环境
  • 组织知识与规则
P1

准备中国市场语言

  • 专属版
  • 私有部署
  • 模型可替换
  • 内网与审计能力

候补样本与不宜硬抄的部分

这些样本不是没价值，而是当前更适合放在观察区，不应挤占主报告里的高优先级席位。

候补

文心快码

ToB 气质很强，尤其适合拆企业控制台与政企案例包装，但和通义灵码、Trae 的重叠也很高。

候补

Aider

终端工作流的经典基线，开源影响力强，但 seat 经济和企业包装不足，不适合放进核心商业样本。

候补

Manus / Bolt

热度和叙事都很强，但公开、稳定、可独立验证的企业证据密度不如正式样本。

候补

Continue / CodeGeeX / MarsCode

一个强在 PR/CI 治理层，一个强在私有化适配，一个强在免费入口意义，但都不足以升格成这一版的核心商业样本。

别盲抄

只学“最火”没意义

如果没有采购、治理、部署、案例和 ROI，只学“最火 agent 产品的故事”并不会变成一门好生意。
